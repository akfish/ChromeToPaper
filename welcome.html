Instapaper Updated

Congratulations! You've installed the latest version of the Instapaper extension. The Options Page has been streamlined to include the authentication process.


News Center:

V2.1: I'd like to aplogize for the quick updates. Version 2.0 was a major break through, but it was not without it's flaws. One user, 2010 politely informed me that there was no feedback that the extension button has been clicked until the page is saved to Instapaper. I realize that it is annoying to be left in the dark, so I've rushed out this update to resolve this issue. When a user presses the Instapaper button, a "..." appears on the badge with a neutral color to let the user know that it is working. Another user, Andre, has given me a good idea. Now that I've implemented it, on the New Tab Page of Chrome, when you click on the Instapaper button, it should bring you straight to the Instapaper site. This was a small change, but should improve the usability of this extension.

V2.0: With this breakthrough version, the extension interface has been revolutionized. Instead of utilizing a Chrome popup bubble, the new extension utilizes color-coded badges to convey the status. Green means success, red means failure, and a grey-ish blue tint indicates a neutral state, neither good nor bad. With this new update, the Options page has been updated to accomodate a host of changes. Now that the login interface has been moved to the options page, a user's credentials are automatically rechecked every time the user accesses the options page. If an issue occurs with the credentials, the extension relaunches the authentication process.
